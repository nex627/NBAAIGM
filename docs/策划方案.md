# NBA 交易模拟器 — 策划方案

## 项目定位

中文版 NBA 交易模拟器，完整 CBA 规则引擎 + AI 交易分析。

- 对标产品：ESPN Trade Machine（英文、纯规则、无 AI）
- 差异化：中文界面 + 代码级规则引擎 + AI 解释 + 移动端优先

## 核心功能

### 1. 交易模拟

- 选择两支球队
- 从每队选择 1-3 名球员 + 选秀权 + 现金
- 实时校验交易规则（薪资匹配、阵容限制、选秀权限制等）
- 判定：✅ 规则通过 / ❌ 具体因哪条规则不通过

### 2. AI 交易分析

- 交易评级（A-F）
- 每队得失分析
- 中文自然语言解释
- 不替代规则引擎，只负责解释

### 3. 移动端优先

- 参考 ESPN Trade Machine 界面布局
- 响应式设计，移动端为主要目标

## 技术架构

```
前端：Next.js + TypeScript + Tailwind CSS
规则引擎：纯 TypeScript（不依赖 AI 判断）
AI 层：LLM API（deepseek/gpt-4o-mini）
数据层：静态 JSON（爬取后生成）
部署：Vercel + 个人 Token
```

## 数据采集方案

### 规则数据源

| 源 | URL | 爬取目标 |
|----|-----|----------|
| cbaguide.com | `cbaguide.com/transactions/trades/tradesalary/` | 薪资匹配规则 |
| cbaguide.com | `cbaguide.com/transactions/trades/tpe/` | 交易特例规则 |
| cbaguide.com | `cbaguide.com/thresholds/salarycap/` | 工资帽阈值 |
| cbafaq.com | `cbafaq.com` | 补充规则细节 |

### 球员/合同数据源

| 源 | URL | 爬取目标 |
|----|-----|----------|
| basketball-reference | `basketball-reference.com/contracts/players.html` | 全联盟 530 份合同 |
| spotrac.com | `spotrac.com/nba` | 球队薪资空间、奢侈税状态 |

### 数据字段

**球员（players.json）**：
```
name / team / salary_2425 / salary_2526 / salary_2627 / 
position / contract_type / option_year / guaranteed / 
trade_kicker / no_trade_clause / trade_eligibility_date
```

**球队（teams.json）**：
```
name / total_payroll / cap_space / tax_status / apron_level /
trade_exceptions / draft_picks_future
```
