# NBA 交易智能体架构设计：球队动机推理层

创建日期：2026-05-24

---

## 1. "Skill" 在这里指什么？

在智能体架构里， skill 通常是一个 可被调用的独立能力单元 ，比如一个工具函数、一段专门的知识库，或者一个带特定提示词的子代理。

如果给 30 支球队每队做一个 skill，最终会变成 30 个高度定制的模块。 技术上可行，但维护成本高，且团队协作困难 。更致命的是，球队的状态（重建/争冠）、薪资空间、总经理偏好都是 动态变化 的，硬编码在 skill 里会让更新变得非常痛苦。

---

## 2. 更优的设计：用"球队状态档案" + 统一"动机推理层"

我建议不要按球队拆 skill，而是建立一个 结构化的球队知识库 ，并让智能体在处理任何交易前，都先 主动查询相关球队的档案 。这样既保持了引擎本身的通用性，又能注入实时、真实的球队动机。

### 具体做法：

#### 第一步：为每支球队定义一份结构化档案

这份档案可以用 JSON 维护，随时更新。核心字段可以包含：

- 基础身份 ：球队名称、所在城市、老板财力（1-10分）、球市大小
- 建队阶段 ：争冠窗口期 / 季后赛级 / 重建积累 / 摆烂
- 核心资产 ：不可交易球员列表、基石型选秀权
- 当前痛点 ：最需要补强的位置、想清理的合同类型
- 薪资哲学 ：是否愿意缴税、愿意缴多少、对硬帽的容忍度
- 总经理风格 ：交易偏好（保守/激进）、选秀权重视程度、是否喜欢做多方大交易、对即战力/潜力的权重

**例如，雷霆的档案可以标记为：** 争冠窗口期开启、握有海量选秀权、珍惜长期灵活性但愿意为明确补强付出资产、总经理普雷斯蒂极度精打细算。

**而太阳的档案则是：** 争冠最后窗口、薪资爆炸已超第二土豪线、交易筹码枯竭、只能做极限的小修小补或重组。

#### 第二步：把"球队情报查询"做成一个 skill（工具）

创建一个 通用的球队状态查询工具 （而非每队一个），比如 `get_team_profile(team_name)`。

智能体在接到交易任务时，第一步就是调用这个工具，获取所有涉及球队的完整档案。这样，球队的动机数据就和交易规则解耦了，你可以单独维护一个 `team_profiles.json` 文件，随时调整个别球队的参数而不碰逻辑。

#### 第三步：在系统提示词里注入"动机推理指令"

在你已有的系统提示词中加入这样的部分，要求模型必须结合档案做动机分析：

```
**球队动机与建队逻辑（极高优先级）**

在执行任何交易建议或评估之前，你必须先通过 `get_team_profile` 获取所有相关球队的当前档案，并严格按以下逻辑思考：

- 如果你是重建队，你的目标是收集年轻资产、选秀权和到期合同，**绝不**送出不受保护的首轮签去换30岁以上的老将。
- 如果你是争冠窗口队，你愿意为明确的短板送出首轮末的选秀权和潜力新秀，但必须谨慎对待未来资产，避免在窗口关闭后陷入泥潭。
- 根据球队的薪资哲学，判断一笔交易增加的奢侈税是否会让老板无法接受。如果球队档案中标记为"拒绝缴税"，则任何让球队进入奢侈税的交易都必须被拒绝。
- 总经理风格会影响交易偏好：精算型会要求获得超值回报，激进型可能愿意小亏补强，恋旧型可能拒绝交易自家培养的明星。
- 你的分析必须明确引用球队档案中的条目，例如："因为[球队A]的档案显示其为'重建积累'阶段，且总经理'极度重视选秀权'，所以送出2030年首轮的行为不合理。"
```

这样， 规则检查器 负责薪资匹配、硬帽等硬约束， 动机推理层 则利用动态档案来驱动"为什么要做这笔交易"的软逻辑，两者分离但协同。

---

## 3. 为什么这比"每队一个 skill"更好？

- **可维护性** ：换一个总经理、球队突然重建，只需改一个 JSON 字段，无需重写整个 skill。
- **一致性** ：所有球队共享同一套推理逻辑，不会出现"湖人 skill 忘记写硬帽规则"的情况。
- **可扩展** ：未来你想加"教练体系"偏好（比如德安东尼只要跑轰球员）、伤病影响等，只需在档案里加新字段，工具和推理指令微调即可。

---

## 4. 你可以立刻开始的实操步骤

1. **先维护一份 `team_profiles.json`** ，为30支球队填好上述字段。初始版本可以粗糙些，后面慢慢校准。
2. **在你的智能体框架中注册一个工具函数 `get_team_profile`** ，接收球队全称或缩写，返回对应档案文本。
3. **修改系统提示词** ，加入我上面给的"动机推理指令"，要求模型在生成或评估任何交易前必须主动调用该工具，并让动机分析成为输出的一部分。
4. **测试** ：拿一笔明显违背球队动机的交易（比如让活塞送出坎宁安去换霍勒迪），看智能体是否会主动拒绝并指出活塞正处于重建、总经理需要年轻核心。

---

## 总结

不要做成 30 个 skill。 用"一个查询工具 + 一份可维护的球队档案数据库 + 一套动机推理提示词" ，就能让智能体在规则之外，拥有真正的"球队大脑"。这远比蒸馏出 30 个死板的小模型灵活且强大。
