# NBA 交易模拟器 — AI 功能规划

> 创建日期：2026-05-24 | 关联文档：[策划方案.md](./策划方案.md) [DEVELOPMENT_PLAN.md](./DEVELOPMENT_PLAN.md)

---

## 一、设计原则

| 原则 | 说明 |
|------|------|
| AI 不判规则，只解释规则 | 规则引擎是权威，AI 是翻译官 |
| AI 建议必须过规则引擎 | AI 生成的交易方案必须二次校验，不通过自动过滤 |
| 流式输出 | AI 分析结果流式展示，用户无需等待完整响应 |
| 上下文窗口优化 | 只传必要数据给 LLM，控制 token 成本 |
| 离线降级 | AI 服务不可用时，规则引擎照常工作 |

## 二、AI 功能分层架构

```
┌─────────────────────────────────────────────────────┐
│  Layer 3: AI 创造层 — 主动生成，引导用户              │

├─────────────────────────────────────────────────────┤

│  (规则白话解释 / 交易结果解读)                        │
└─────────────────────────────────────────────────────┘
```

## 三、Layer 1: AI 基础层（MVP 必做）

### 3.1 交易结果解读

**触发时机**：用户提交交易 → 规则引擎校验完成 → AI 解读结果

**功能描述**：
- 规则引擎给出 ✅/❌ 后，AI 用中文自然语言解释"为什么通过/不通过"
- 不替代规则引擎，只负责翻译规则引擎的输出

**输入**：
```json
{
  "trade": { "teams": [...], "moves": [...] },
  "ruleResults": [
    { "name": "湖人·薪资匹配", "passed": false, "desc": "送出 $52M → 获得 $65M | 上限 $65.35M（税队 ×1.25+$100K）| 超出 $0" },
    { "name": "湖人·阵容人数", "passed": true, "desc": "15人 → 14人" }
  ]
}
```

**输出示例**：
```
🤖 AI 解读：
这笔交易在薪资匹配上勉强过关。湖人送出 LeBron James（$52.6M），
换回两名球员合计 $65M，刚好卡在税队 ×1.25+$100K 的上限内。
但要注意：交易后湖人薪资将超过第二土豪线，将失去以下权限：
- 不能使用聚合特例（Aggregated TPE）
- 不能在买断市场签人
- 未来首轮签可能被冻结
阵容方面，湖人从 15 人变成 14 人，还有 1 个名额可以补强。
```

**技术方案**：
- 将规则引擎结果 + 交易数据打包为 prompt
- 调用 LLM API（deepseek / gpt-4o-mini）
- 流式输出到前端

**AI 轮次**：1 轮

---

### 3.2 CBA 规则白话解释

**触发时机**：用户点击规则名称旁的 ❓ 图标

**功能描述**：
- 将 CBA 法律条文翻译成球迷能懂的白话
- 结合当前交易场景举例说明

**输出示例**：
```
用户点击 "薪资匹配规则" 旁的 ❓

🤖 什么是薪资匹配？
NBA 规定交易中每队送出和接收的薪资必须大致对等。
具体来说：
- 帽下队（有薪资空间的队）：可接收送出薪资的 2 倍 + $250K
- 税下队（超帽但未超税线）：可接收送出薪资的 1.25 倍 + $250K
- 超税队：可接收送出薪资的 1.25 倍 + $100K
- 第二土豪线队：只能接收送出薪资的 100%（1:1 等额交换）

以你这笔交易为例，湖人超了税线，所以上限是送出薪资 ×1.25+$100K。
```

**技术方案**：
- 预置 CBA 规则知识库（已有 rules.json）
- 检索相关规则段落 + 用户当前交易上下文
- LLM 生成白话解释

**AI 轮次**：1 轮

## 四、Layer 2: AI 深度层（核心差异化）

### 4.1 交易评级（A-F）

**触发时机**：交易规则通过后，自动生成评级

**功能描述**：
- 对每支参与交易的球队分别评级
- 综合评级（整体交易公平性）

**评级维度**：

| 维度 | 权重 | 说明 |
|------|------|------|
| 战力变化 | 40% | PER/Win Share 变化 |
| 薪资灵活性 | 25% | 帽空间/税线位置变化 |
| 未来资产 | 20% | 选秀权得失、年轻球员潜力 |
| 阵容适配 | 15% | 位置重叠、化学反应 |

**输出示例**：
```
🤖 AI 交易评级

湖人：C+
  战力：B  → 得到两名即战力，但失去核心
  薪资：D  → 深陷第二土豪线，灵活性极差
  未来：C  → 送出 2028 首轮，未来资产缩水
  适配：B- → 位置互补，但球权分配存疑

凯尔特人：A-
  战力：A  → 得到超级巨星，即战力飙升
  薪资：B  → 释放空间，避开第二土豪线
  未来：B+ → 付出首轮但得到建队核心
  适配：A  → 完美补强短板

综合评级：B（一方明显获利，但规则上可行）
```

**技术方案**：
- 球员数据（PER、薪资、位置）+ 交易结构作为 prompt
- LLM 生成评级 + 分析
- 可选：本地计算基础评分，LLM 负责润色和解释

**AI 轮次**：1-2 轮

---

### 4.2 球员对比分析

**触发时机**：交易中选中球员时，展示对比面板

**功能描述**：
- 交易双方球员的直观对比
- AI 生成"谁更值得拥有"的分析

**输出示例**：
```
🤖 球员对比

LeBron James vs Anthony Davis + Klay Thompson

          LeBron    Davis    Thompson
年龄        41        32        35
PER       15.0      10.0      11.0
薪资     $52.6M    $54.1M    $16.7M
剩余年限    2年       3年       2年

AI 分析：
单看数据，LeBron 的 PER 仍高于 Davis 和 Thompson 之和。
但考虑到薪资占比（LeBron 一人占湖人薪资 38%），换回 Davis+Thompson
可以让阵容更有深度。不过 Davis 的伤病史是最大风险。
```

**AI 轮次**：1 轮

---

### 4.3 历史交易类比

**触发时机**：交易评级生成后，附加展示

**功能描述**：
- 找到历史上类似的交易案例
- 对比结果，帮助用户判断交易合理性

**输出示例**：
```
🤖 历史类比

这笔交易类似于 2022 年篮网送出 KD 换回 Mikal Bridges + 4 个首轮：
- 都是超级巨星换回多个资产
- 篮网那笔交易后来被评为 A+（Bridges 成长为全明星）
- 但 KD 的交易价值远高于当前这笔中的球员

参考结论：巨星换深度资产，短期战力下降但长期可能更赚。
```

**技术方案**：
- 预置 50-100 个经典交易案例作为知识库
- RAG 检索 + LLM 生成类比分析

**AI 轮次**：1 轮

## 五、Layer 3: AI 创造层（高级功能）

### 5.1 AI 交易建议

**触发时机**：用户选择一支球队后，点击"💡 AI 建议交易"

**功能描述**：
- 基于球队现状（薪资、阵容、位置需求），AI 主动推荐可行交易
- 每次推荐 3 个方案，按可行性排序

**输出示例**：
```
🤖 AI 为湖人推荐交易

方案 1：湖人 ↔ 公牛
  送出：Rui Hachimura ($18.3M) + Jarred Vanderbilt ($11.6M)
  换回：Zach Collins ($18.1M) + Patrick Williams ($18.0M)
  规则：✅ 薪资匹配通过
  评级：B+
  理由：湖人获得更年轻的锋线，公牛得到到期合同释放空间

方案 2：湖人 ↔ 活塞
  送出：Maxi Kleber ($11M) + Luke Kennard ($11M)
  换回：Caris LeVert ($14.1M) + Paul Reed ($5.3M)
  规则：✅ 薪资匹配通过
  评级：B-
  理由：湖人补强后卫深度，活塞获得到期合同

方案 3：湖人 ↔ 篮网（三方交易）
  ...三方交易方案...
```

**技术方案**：
- 球队数据 + 规则引擎约束作为 prompt
- LLM 生成交易方案
- 关键：AI 生成的方案必须经过规则引擎二次校验，不通过的自动过滤
- 流程：AI 生成 → 规则引擎校验 → 过滤/调整 → 展示

**AI 轮次**：2-3 轮

---

### 5.2 球队薪资顾问

**触发时机**：用户在球队详情页点击"🧠 AI 顾问"

**功能描述**：
- 分析球队薪资结构
- 给出避税/补强建议
- 模拟不同操作后的薪资变化

**输出示例**：
```
🤖 湖人薪资顾问

当前状态：
  总薪资：$189.7M（超第一土豪线 $189.0M）
  距第二土豪线：$12.3M
  奢侈税预估：$8.5M

⚠️ 风险提示：
  目前超过第一土豪线，触发了硬帽限制。
  如果再超过第二土豪线，将面临首轮签冻结风险。

💡 AI 建议：
  1. 交易 Maxi Kleber ($11M) → 释放空间回到税线下
  2. 用底薪特例签自由球员 → 不影响薪资匹配
  3. 避免使用 NTMLE → 会触发第一土豪线硬帽
```

**AI 轮次**：1-2 轮

---

### 5.3 交易场景生成器

**触发时机**：首页"🎲 随机交易"按钮

**功能描述**：
- 随机生成有趣的交易场景
- 类似"每日一题"的互动玩法
- 社交分享友好

**输出示例**：
```
🎲 今日挑战

如果勇士想同时得到 Giannis 和 Lillard，需要送出谁？

AI 生成的方案：
  勇士送出：Stephen Curry + Jimmy Butler + 2028 首轮
  雄鹿送出：Giannis Antetokounmpo
  开拓者送出：Damian Lillard

  规则校验：❌ 薪资不匹配（勇士送出 $113.5M，接收 $68.3M）

  你能找到让这笔交易通过的方案吗？试试看！
```

**AI 轮次**：1 轮

## 六、技术架构

```
│        │             │             │             │           │
│        ▼             ▼             ▼             ▼           │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              AI Service Layer (TypeScript)           │    │
│  │  ┌─────────────┐  ┌──────────────┐  ┌───────────┐  │    │

              ┌─────────────────────┐
              │   LLM API           │
              │   deepseek / gpt-4o │
              │   (客户端直连)       │
              └─────────────────────┘
```

### 6.2 AI 调用流程

```
用户操作 → 构建上下文 → 选择 Prompt 模板 → 调用 LLM API → 流式渲染
    │           │              │                  │              │
    │     交易数据          规则解释/评级/      deepseek-chat    逐字展示
    │     规则结果          建议/对比          或 gpt-4o-mini   loading 态
    │     球员数据          对应不同模板
    │
    └── AI 建议特殊流程 ──→ AI 生成方案 → 规则引擎二次校验 → 过滤/调整 → 展示
```

### 6.3 LLM 选型

| 用途 | 模型 | 理由 |
|------|------|------|
| 规则解释 / 交易解读 | deepseek-chat | 成本低，中文好，够用 |
| 交易评级 / 建议生成 | deepseek-chat / gpt-4o-mini | 需要推理能力 |
| 历史类比 | gpt-4o-mini | 需要知识广度 |

### 6.4 Prompt 模板设计

每个 AI 功能对应一个 Prompt 模板，模板结构：

```
系统提示（角色定义 + 输出格式约束）
  +
用户上下文（交易数据 / 规则结果 / 球员数据）
  +
输出示例（few-shot 示例，保证格式一致）
```

模板存放位置：`src/lib/ai/prompts/`

```
src/lib/ai/
├── prompts/
│   ├── trade-explainer.ts      ← 交易结果解读
│   ├── rule-explainer.ts       ← CBA 规则白话解释
│   ├── trade-grader.ts         ← 交易评级
│   ├── player-compare.ts       ← 球员对比
│   ├── trade-suggester.ts      ← AI 交易建议
│   ├── salary-advisor.ts       ← 薪资顾问
│   └── scenario-generator.ts   ← 场景生成器
├── context-builder.ts          ← 上下文构建器
├── llm-client.ts               ← LLM API 客户端
└── stream-renderer.ts          ← 流式输出渲染
```

## 八、成本估算

### 8.1 Token 用量估算

| 功能 | 单次调用 Token | 日均调用次数 | 日均 Token |
|------|---------------|-------------|-----------|
| 交易结果解读 | ~1,500 | 200 | 300K |
| CBA 规则解释 | ~1,000 | 100 | 100K |
| 交易评级 | ~2,000 | 150 | 300K |
| 球员对比 | ~1,500 | 80 | 120K |
| AI 交易建议 | ~3,000 | 50 | 150K |
| **合计** | | | **~970K/日** |

### 8.2 月成本估算（deepseek-chat）

- deepseek-chat 定价：输入 ¥1/M token，输出 ¥2/M token
- 日均 ~1M token，月均 ~30M token
- 预估月成本：¥60-90（初期用户量少，成本极低）

### 8.3 限流策略

| 策略 | 实现 |
|------|------|
| 用户级限流 | localStorage 计数，每日 20 次 AI 调用 |
| 功能级限流 | 交易建议每次最多 3 个方案 |
| 降级方案 | AI 不可用时展示规则引擎原始结果 |
| 缓存 | 相同交易方案缓存 AI 结果（hash 交易结构） |

## 九、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| LLM 幻觉（给出错误规则解释） | 用户误解 CBA 规则 | 所有 AI 输出标注"仅供参考"，规则引擎结果为权威 |
| API 不可用 | AI 功能中断 | 降级到纯规则引擎模式，核心功能不受影响 |
| Token 成本超预期 | 运营成本上升 | 限流 + 缓存 + 选用更便宜模型 |
| AI 交易建议不合规 | 推荐了规则不允许的交易 | 强制二次校验，不通过不展示 |
| 响应速度慢 | 用户体验差 | 流式输出 + 骨架屏 + 超时降级 |
