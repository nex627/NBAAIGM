# AI数据准确性保障机制

## 🏗️ 架构原则

### 核心原则：爬虫数据是唯一真相源

本系统是**真实交易模拟器**，所有动态数据（球员名单、薪资、合同状态、球队归属）均来自爬虫系统实时获取的真实数据。

```
爬虫系统 (contracts.json / rules.json)
        ↓
  gen_data.js (数据生成脚本)
        ↓
  data.js (前端数据) + team-profiles/*.md (球队档案)
        ↓
  AI 提示词 (注入球队数据 + 球队档案)
        ↓
  LLM 输出 (严格基于注入数据)
```

### 数据流设计

| 数据类型 | 来源 | 更新方式 | AI是否可修改 |
|----------|------|----------|:---:|
| 球员名单/薪资 | 爬虫 → gen_data.js → data.js | 重新运行爬虫+生成脚本 | ❌ |
| CBA规则 | 爬虫 → rules.json | 重新运行爬虫 | ❌ |
| 球队策略分析 | team-profiles/*.md | 人工维护 | ❌ |
| 交易建议 | LLM 生成 | 实时 | ✅ (但必须过规则引擎) |

### 关键设计决策

1. **无剧情修正层**：gen_data.js 中不包含任何 TEAM_CORRECTIONS 或类似的硬编码覆盖，球员所属球队完全由爬虫数据决定
2. **球队档案只含客观分析**：team-profiles 只保留建队策略、球员技术特点、管理层风格等客观信息，不包含虚构赛季战绩、球员离队剧情等
3. **AI 提示词强制数据真实性**：所有 AI 系统提示词都包含"数据真实性原则"声明，要求 AI 只使用系统提供的数据

## ✅ 当前保障措施

### 1. 数据层：爬虫数据唯一真相源

**数据生成流程**：
```
contracts.json (爬虫数据) → gen_data.js → data.js (前端数据)
```

**关键约束**：
- gen_data.js 不包含任何硬编码的球队分配覆盖
- 球员所属球队完全由 `contracts.json` 中的 `team` 字段决定
- 任何数据修正必须回到爬虫源头修改，不能在生成脚本中覆盖

### 2. 球队档案层：只含客观分析

**档案结构**（7 个维度）：
1. 建队阶段与战略方向
2. 核心球员特点分析（技术特点 + 球队定位 + 交易价值）
3. 总经理决策风格
4. 老板意愿与投资哲学
5. 教练体系与战术偏好
6. 薪资状况与约束
7. 选秀权状况

**禁止内容**：
- ❌ 虚构赛季战绩（如"32胜50负东部第11"）
- ❌ 虚构球员离队剧情（如"已告知球队希望离队"）
- ❌ 虚构伤病剧情（如"以伤病为由关机"）
- ❌ 虚构交易传闻（如"休赛期极可能被交易"）

**数据来源声明**（每个档案必须包含）：
```
> ⚠️ 数据来源声明：球员名单、合同状态等动态数据以爬虫系统实时获取的真实数据为准，本档案仅提供技术特点和球队定位分析。
```

### 3. 提示词层：数据真实性原则

**所有 AI 系统提示词都包含以下声明**：

```
⚠️ 数据真实性原则（最高优先级）：
- 本系统是真实交易模拟器，所有球员数据来自爬虫系统实时获取的真实数据，爬虫数据是唯一真相源
- 球员所属球队、薪资、合同状态等以系统提供的数据为准，绝不使用训练知识修正
- 不要假设任何球员已离队、要求交易或受伤，除非系统数据明确标注
- 绝不要编造球员交易去向、签约情况或阵容变化
```

**已添加声明的提示词文件**：
- `prototype/js/ai-service.js`: TRADE_ADVISOR_SYSTEM, TRADE_EXPLAINER_SYSTEM, GM_CHAT_SYSTEM
- `src/lib/ai/prompts/trade-advisor.ts`
- `src/lib/ai/prompts/trade-grader.ts`
- `src/lib/ai/prompts/gm-chat.ts`
- `src/lib/ai/prompts/salary-advisor.ts`
- `src/lib/ai/prompts/player-compare.ts`
- `src/lib/ai/prompts/rule-explainer.ts`
- `src/lib/ai/prompts/trade-explainer.ts`
- `src/lib/ai/prompts/trade-analogy.ts`
- `src/lib/ai/prompts/scenario-generator.ts`

### 4. 架构层：规则引擎校验

- AI 生成的交易方案必须经过规则引擎二次校验
- 不通过的交易方案自动过滤
- AI 不判规则，只解释规则

## 🛡️ 数据维护流程

### 更新球员数据

```
运行爬虫 → 更新 contracts.json / rules.json → 运行 gen_data.js → 验证 data.js
```

```bash
# 1. 运行爬虫获取最新数据
cd data/scripts && node crawler.js

# 2. 重新生成 data.js
node gen_data.js

# 3. 验证数据一致性
node check-data-consistency.js
```

### 更新球队档案

球队档案中的策略分析需要人工维护，更新时遵循：
1. 只更新客观分析（建队策略、球员特点、管理层风格）
2. 不添加任何虚构剧情
3. 动态数据（球员名单、薪资）由爬虫系统提供，档案中不重复记录
4. 每个章节标注更新频率和上次验证日期
